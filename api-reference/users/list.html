List Users

Retrieve a list of users with pagination support

Returns a list of users for your account. The users are returned sorted by creation date, with the most recent users appearing first.

​
Query Parameters

​
limit
integer
default:"10"
A limit on the number of objects to be returned. Limit can range between 1 and 100, and the default is 10.
​
starting_after
string
A cursor for use in pagination. starting_after is an object ID that defines your place in the list. For instance, if you make a list request and receive 100 objects, ending with usr_xyz, your subsequent call can include starting_after=usr_xyz in order to fetch the next page of the list.
​
ending_before
string
A cursor for use in pagination. ending_before is an object ID that defines your place in the list. For instance, if you make a list request and receive 100 objects, starting with usr_abc, your subsequent call can include ending_before=usr_abc in order to fetch the previous page of the list.
​
email
string
Filter users by email address. Supports partial matching.
​
status
string
Filter by user status. Options: active, inactive, pending
​
created_after
string
Only return users created after this timestamp (ISO 8601 format).
​
created_before
string
Only return users created before this timestamp (ISO 8601 format).

Pagination

This endpoint uses cursor-based pagination. Use the starting_after and ending_before parameters to navigate through pages.

​
Example: Paginating Forward

Copy
# Get first page
response = requests.get(
    'https://api.example.com/v1/users',
    headers={'Authorization': 'Bearer YOUR_API_KEY'},
    params={'limit': 10}
)
first_page = response.json()

# Get next page using last user's ID
last_user_id = first_page['data'][-1]['id']
response = requests.get(
    'https://api.example.com/v1/users',
    headers={'Authorization': 'Bearer YOUR_API_KEY'},
    params={'limit': 10, 'starting_after': last_user_id}
)
next_page = response.json()
For best performance, use a limit of 100 or less. Larger page sizes may result in slower response times.
