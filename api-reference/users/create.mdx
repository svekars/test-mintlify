Creates a new user object with the specified parameters.

## Request Body

<ParamField body="email" type="string" required>
  The user's email address. Must be unique across all users.
</ParamField>

<ParamField body="name" type="string" required>
  The user's full name.
</ParamField>

<ParamField body="password" type="string">
  The user's password. If not provided, a temporary password will be generated and sent via email.
</ParamField>

<ParamField body="status" type="string" default="pending">
  Initial user status. Options: `active`, `inactive`, `pending`
</ParamField>

<ParamField body="metadata" type="object">
  Set of key-value pairs that you can attach to a user. Useful for storing additional information about the user in a structured format.
</ParamField>

<ParamField body="send_welcome_email" type="boolean" default="true">
  Whether to send a welcome email to the user.
</ParamField>

## Response

Returns the created user object if successful. Returns an error if the request parameters are invalid or the email is already in use.

<ResponseField name="id" type="string">
  Unique identifier for the user
</ResponseField>

<ResponseField name="email" type="string">
  User's email address
</ResponseField>

<ResponseField name="name" type="string">
  User's full name
</ResponseField>

<ResponseField name="status" type="string">
  User status
</ResponseField>

<ResponseField name="created_at" type="string">
  Timestamp of when the user was created
</ResponseField>

<ResponseField name="updated_at" type="string">
  Timestamp of when the user was last updated
</ResponseField>

<ResponseField name="metadata" type="object">
  Additional information attached to the user
</ResponseField>

## Validation Rules

<AccordionGroup>
  <Accordion title="Email">
    - Must be a valid email format
    - Must be unique across all users
    - Maximum length: 255 characters
    - Cannot be changed after creation
  </Accordion>

  <Accordion title="Name">
    - Required field
    - Minimum length: 2 characters
    - Maximum length: 100 characters
  </Accordion>

  <Accordion title="Password">
    - Minimum length: 8 characters
    - Must contain at least one uppercase letter
    - Must contain at least one lowercase letter
    - Must contain at least one number
    - Optional special characters recommended
  </Accordion>

  <Accordion title="Metadata">
    - Maximum of 50 key-value pairs
    - Keys must be strings (max 40 characters)
    - Values must be strings (max 500 characters)
    - Cannot contain nested objects
  </Accordion>
</AccordionGroup>

## Best Practices

<Tip>
  **Use metadata for custom fields**: Store additional user information in the metadata field rather than creating custom fields. This provides flexibility without schema changes.
</Tip>

<Warning>
  **Rate limiting**: User creation is rate-limited to 100 requests per minute. If you need to create users in bulk, consider using the batch creation endpoint instead.
</Warning>

<Note>
  **Email verification**: Users created with status `pending` will need to verify their email address before they can log in. An email with a verification link is automatically sent.
</Note>
